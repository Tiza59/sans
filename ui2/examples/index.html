Introduction

Sans UI is a universal UI framework that maps Web Components and Svelte to NodeGUI (Qt) for desktop and NativeScript (UIKit/Android Views) for mobile.

This allows developers to write UI once using Web Component/Svelte syntax and have it render as native widgets on Windows, macOS, Linux, iOS, and Android.

Core Architecture

The Sans UI architecture consists of several key components:

  • Web Components & Svelte as the API Layer

    Define UI using Web Components or Svelte syntax with a consistent API.

  • Platform Detection

    Automatically detects the current platform (desktop, mobile, or web).

  • Native UI Adapters

    Maps web components to native UI widgets on each platform.

  • Event System

    Translates web events to native platform events.

Component Mapping

Sans UI provides a standard UI schema that maps to both NodeGUI and NativeScript:

Component Web API Desktop (NodeGUI/Qt) Mobile (NativeScript)
Button <sans-button> QPushButton Button
Text Input <sans-textbox> QLineEdit TextField
Label <sans-label> QLabel Label
List View <sans-list> QListWidget ListView
Image <sans-image> QPixmap Image
Container <sans-container> QWidget StackLayout
